Helps in selecting test cases by focussing on the boundary values of input variables or conditions, as theyse values tend to be more error prone.

BVA is usually combined with equivalence partitioning.

Characteristics of BVA:
- Proves the points at which the rules changes have been encoded are correct
- For each boundary marker, three boundary values might be tested.

Done because of things like "off by one error", whether that's a logic or spec bug.
